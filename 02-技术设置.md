# 1. 文档目的

本文档用于指导 RealtimeASR SDK 开发。

阅读本文档后，开发人员应能够：

- 理解整体架构
- 理解模块职责
- 理解数据流
- 按照统一规范完成开发
- 保证模块之间低耦合

本文档不介绍产品功能，仅介绍技术实现。

---

# 2. 整体架构

SDK 采用经典分层架构。

```

Business

↓

RealtimeASR

↓

Recorder

↓

Audio Engine

↓

AudioWorklet

↓

PCM

↓

Provider

↓

Tencent ASR

```

所有模块均采用单向依赖。

禁止循环依赖。

---

# 3. 包结构

```

packages/

core/

provider-tencent/

shared/

types/

demo/

```

其中：

```

core

↓

provider

↓

shared

```

Core 不允许依赖 Provider。

---

# 4. Core 模块

Core 是 SDK 最核心部分。

职责：

- 管理录音
- 管理 AudioContext
- PCM 输出
- 状态管理
- 生命周期
- EventEmitter

Core 不知道：

- 腾讯
- 阿里
- FunASR

Core 永远不知道当前使用哪个 Provider。

---

# 5. Class 设计

## Recorder

入口类。

职责：

- 初始化
- start()
- stop()
- destroy()
- 管理状态
- 派发事件

禁止：

- WebSocket
- 签名
- JSON解析

Recorder 不负责任何 Provider。

---

### 属性

```

private state

private provider

private audioEngine

private frameBuffer

private emitter

```

---

### 方法

```

constructor()

start()

stop()

destroy()

```

---

Recorder 生命周期：

```

Idle

↓

Initializing

↓

Recording

↓

Stopping

↓

Stopped

↓

Destroyed

```

---

## AudioEngine

职责：

负责浏览器音频。

内部：

```

MediaStream

↓

AudioContext

↓

AudioWorkletNode

```

提供：

```

init()

start()

stop()

destroy()

```

---

AudioEngine 不允许：

- 上传数据
- WebSocket

---

## AudioWorkletManager

职责：

加载：

```

processor.js

```

负责：

```

register()

connect()

disconnect()

```

输出：

```

Float32Array

```

---

主线程收到：

```

Float32Array

```

后立即交给：

```

PCMEncoder

```

---

## PCMEncoder

职责：

Float32

↓

PCM16

```

Float32

↓

Int16Array

```

不得负责：

- Buffer
- 上传

---

方法：

```

encode()

```

---

## Resampler

输入：

```

48000Hz

```

输出：

```

16000Hz

```

算法：

后续采用：

Linear Interpolation。

要求：

不得依赖 Provider。

---

方法：

```

resample()

```

---

## FrameBuffer

职责：

累计 PCM。

腾讯：

```

200ms

```

默认：

```

3200 Sample

↓

6400 Byte

```

Buffer 满：

立即：

```

provider.send()

```

不得缓存无限增长。

采用 RingBuffer。

---

方法：

```

push()

clear()

flush()

```

---

## VolumeMeter

输入：

```

Float32Array

```

输出：

```

0~100

```

算法：

```

RMS

```

方法：

```

calculate()

```

---

## EventEmitter

统一事件中心。

不得依赖浏览器 EventTarget。

自行实现。

支持：

```

on()

once()

off()

emit()

```

---

## StateMachine

所有状态流转统一管理。

禁止：

```

Recording

↓

再次 start()

```

必须：

```

throw Error

```

---

# 6. Provider

Provider 是唯一允许访问网络的模块。

Provider 不允许：

- 创建 AudioContext
- 获取麦克风

Provider 只负责：

```

connect()

↓

send()

↓

receive()

↓

close()

```

---

统一接口：

```ts

interface ASRProvider {

connect()

send(buffer)

close()

}

```

---

# 7. TencentProvider

职责：

- URL 签名
- WebSocket
- JSON解析
- 协议转换

内部：

```

Signer

↓

WebSocket

↓

Parser

```

不得直接访问 Recorder。

只能：

```

```

# 19. 最终原则

整个 SDK 必须遵循以下原则：

- Core 永远不依赖 Provider
- Provider 永远不依赖 UI
- UI 永远不依赖实现细节
- Recorder 是唯一入口
- EventEmitter 是唯一通信方式
- StateMachine 是唯一状态管理
- AudioWorklet 是唯一 PCM 来源
- Provider 是唯一网络模块
- Core 永远保持纯净

以上原则任何开发过程中均不得违反。
